# AX Status — Document Health Report

Check whether knowledge documents are still up-to-date with the source code they describe.

## Workflow

### Step 1: Scan Knowledge Files

Find all AX-generated documents:
```
docs/ai-context/*.md
.agents/skills/*/SKILL.md
*/AGENTS.md (module-level)
```

### Step 2: Extract Metadata

For each document, look for the `ax-meta` comment block at the end:
```markdown
<!-- ax-meta
sources:
  - path/to/source1.kt
  - path/to/source2.ts
generated: 2026-04-20
-->
```

If no `ax-meta` block exists, skip that document (it wasn't generated by AX or predates metadata).

### Step 3: Check Freshness

For each document with metadata:
1. Read the `generated` date
2. For each file in `sources`, check its last modification time via `git log -1 --format=%ai -- <path>`
3. If any source file was modified after the `generated` date, mark the document as stale

### Step 4: Report

Output a health report table:

```
Document Health Report
┌────────────────────────┬────────┬──────────────────┐
│ Document               │ Status │ Details          │
├────────────────────────┼────────┼──────────────────┤
│ architecture.md        │ stale  │ PageManager changed │
│ data-flow.md           │ ok     │                  │
│ debug-diff/SKILL.md    │ stale  │ DiffManager changed │
└────────────────────────┴────────┴──────────────────┘
```

### Step 5: Suggest Actions

For stale documents, suggest:
- "Run `/ax architecture` to update architecture docs"
- "Run `/ax skill debug-diff` to update the debug skill"

For documents with missing source files (source was deleted/renamed):
- Flag as "source missing" and suggest review
